# 风电送出线路巡检后端系统

清洁风电稳定外送，四十多公里送出线路不能只在图纸上躺着。本系统用于送出线路的分段建档、周期巡检、缺陷工单全流程管理和统计分析。

## 技术栈

- **Node.js** + **Express** - 轻量后端服务
- **SQLite (better-sqlite3)** - 嵌入式数据库，数据随项目携带
- **node-cron** - 定时巡检任务生成

## 功能模块

### 1. 线路分段建档
- **线路 (lines)**: 记录送出线路名称、电压等级、全长、起止点、装机容量
- **杆塔 (towers)**: 每基杆塔的海拔、经纬度、道路可达性、覆冰风险、责任班组
- **区段 (segments)**: 两基杆塔之间的区段，平均海拔、可达性、覆冰风险、责任班组、外送容量
- **跨越点 (crossing_points)**: 区段内跨越公路、铁路、河流、其他线路等
- **隐患点 (hazard_points)**: 区段内地质滑坡、覆冰、山火等风险点

### 2. 巡检任务管理
- 按覆冰风险自动设置巡检周期（高风险15天、中风险20天、低风险30天）
- 每日凌晨 6:00 自动生成下一批巡检任务
- 支持手动触发任务生成和临时巡检任务

### 3. 巡检记录上报
巡检员对每基杆塔上报以下检查项：
- **绝缘子**: 状态 + 备注
- **金具**: 状态 + 备注
- **导线弧垂**: 实测值 + 状态 + 备注
- **塔基边坡**: 状态 + 备注
- **通道树障**: 状态 + 备注

异常项目自动生成对应缺陷工单。

### 4. 缺陷工单全流程
三级分级：
- **一般缺陷**: 30天内处理
- **严重缺陷**: 7天内处理，区段外送容量限制到 80%
- **危急缺陷**: 1天内处理，区段外送容量限制到 50%，线路总容量联动调整

流程：`待分配 → 处理中 → 待复核 → 已关闭`（复核不通过回退到处理中）
- 危急/严重缺陷在处理期间自动限制对应区段外送能力
- 缺陷复核关闭后自动恢复外送容量

### 5. 统计分析
- **缺陷密度**: 各区段每公里缺陷数，按严重程度细分
- **平均消缺时长**: 各区段和总体的平均/最大/最小处理天数
- **超期未处理**: 超期工单明细，按区段、责任班组、严重度汇总
- **线路可用率**: 近30天容量受限天数比例，当前容量/额定容量比

## 快速启动

```bash
npm install
npm start
```

服务默认运行在 `http://localhost:3000`，首次启动自动写入种子数据（2条线路、15基杆塔、6个区段、若干巡检记录和缺陷工单）。

手动重写种子数据：
```bash
npm run seed
```

## API 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 系统信息与接口索引 |
| GET/POST | `/api/lines` | 线路列表/新增 |
| GET/PUT/DELETE | `/api/lines/:id` | 线路详情/修改/删除（含杆塔和区段） |
| GET/POST | `/api/towers` | 杆塔列表/新增 |
| GET/PUT/DELETE | `/api/towers/:id` | 杆塔详情/修改/删除 |
| GET/POST | `/api/segments` | 区段列表/新增 |
| GET/PUT/DELETE | `/api/segments/:id` | 区段详情（含跨越点、隐患点） |
| POST | `/api/tasks/generate` | 手动触发巡检任务周期生成 |
| GET/POST | `/api/tasks` | 巡检任务列表/新增 |
| GET/PUT | `/api/tasks/:id` | 任务详情/状态变更 |
| GET/POST | `/api/records` | 巡检记录列表/新增上报 |
| GET/POST | `/api/defects` | 缺陷工单列表/新建 |
| PUT | `/api/defects/:id/assign` | 分配处理人 |
| PUT | `/api/defects/:id/handle` | 提交处理结果 |
| PUT | `/api/defects/:id/review` | 复核关闭（pass:false 退回） |
| GET | `/api/stats/summary` | 系统总体概况 |
| GET | `/api/stats/defect-density` | 各区段缺陷密度 |
| GET | `/api/stats/repair-duration` | 平均消缺时长 |
| GET | `/api/stats/overdue` | 超期未处理统计 |
| GET | `/api/stats/availability` | 线路可用率与容量 |

## 数据文件

SQLite 数据库文件位于 `data/inspection.db`，已在 `.gitignore` 中忽略。首次启动自动写入种子数据。
